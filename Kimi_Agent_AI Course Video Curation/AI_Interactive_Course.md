## Hands-On Exercise: Build a Simple Neural Network

### Exercise 1.1: Your First Neural Network (Python)

```python
# Install required packages:
# pip install numpy tensorflow

import numpy as np
import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

# Step 1: Create a simple dataset
# Problem: Predict whether a number is positive or negative
X = np.array([[-5], [-3], [-1], [0], [1], [3], [5], [10], [-10], [2]])
y = np.array([[0], [0], [0], [0], [1], [1], [1], [1], [0], [1]])  # 0=negative, 1=positive

# Step 2: Build the neural network
model = keras.Sequential([
    layers.Dense(4, activation='relu', input_shape=(1,)),  # Hidden layer with 4 neurons
    layers.Dense(1, activation='sigmoid')  # Output layer (binary classification)
])

# Step 3: Compile the model
model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy'])

# Step 4: Train
print("Training the neural network...")
model.fit(X, y, epochs=200, verbose=0)

# Step 5: Test predictions
test_numbers = np.array([[-7], [4], [0.5], [-2]])
predictions = model.predict(test_numbers, verbose=0)

print("\nPredictions (closer to 1 = positive, closer to 0 = negative):")
for num, pred in zip(test_numbers, predictions):
    label = "positive" if pred[0] > 0.5 else "negative"
    confidence = pred[0] if pred[0] > 0.5 else 1 - pred[0]
    print(f"  {num[0]:>4} → {pred[0]:.4f} → {label} ({confidence:.1%} confidence)")

print("\n--- KEY INSIGHT ---")
print("This tiny network learned to classify numbers without being explicitly programmed!")
print("Real LLMs use the same fundamental principles, just with billions of parameters.")
```

### Exercise 1.2: Understanding Tokens

```python
# pip install tiktoken

import tiktoken

# See how text is broken into tokens (the unit LLMs work with)
encoder = tiktoken.encoding_for_model("gpt-4")

texts = [
    "Hello, world!",
    "Artificial Intelligence",
    "The quick brown fox jumps over the lazy dog",
    "AI",  # Short words may be 1 token
    "antidisestablishmentarianism",  # Long words = many tokens
]

for text in texts:
    tokens = encoder.encode(text)
    print(f"'{text}' → {len(tokens)} tokens: {tokens}")
    # Decode each token individually
    decoded = [encoder.decode([t]) for t in tokens]
    print(f"  Token breakdown: {decoded}\n")
```

## 3.2 Zero-Shot, Few-Shot, Chain-of-Thought

**Key Concepts:**
- **Zero-shot**: No examples, just instructions
- **Few-shot**: Provide 2-5 examples to guide the model
- **Chain-of-Thought (CoT)**: Ask the model to "think step by step"
- **Self-consistency**: Generate multiple answers, pick the most common
- **Automatic CoT**: LLM generates its own reasoning steps

### Example: Chain-of-Thought Prompting

```

## 3.5 ReAct Pattern: Reasoning + Acting

**Key Concepts:**
- ReAct = Reasoning + Acting loop
- The foundation of modern AI agents
- LLM thinks (reasoning) → takes action (tool use) → observes result
- Iterative loop until goal is achieved
- Used by Claude Code, Codex, and most coding agents

### ReAct Loop Pseudocode

```
GOAL: "Find the current weather in Tokyo and suggest clothing"

LOOP (max 10 iterations):
  1. THINK: "I need to find the weather in Tokyo. 
     I have a weather_search tool available."
  2. ACT: Call weather_search(location="Tokyo")
  3. OBSERVE: Result = "Sunny, 28C, light breeze"
  4. THINK: "It's warm and sunny. I should suggest light clothing."
  5. ACT: Finalize response with clothing suggestions
  6. DONE
```

## 3.6 Advanced Techniques

**Tree of Thoughts (ToT):**
- Explore multiple reasoning paths simultaneously
- Like a chess engine evaluating multiple moves
- Best for complex problem-solving

**Graph of Thoughts (GoT):**
- Thoughts form a graph structure (not just a tree)
- Allows merging, revising, and connecting ideas
- Best for creative and research tasks

**Skeleton-of-Thought:**
- Generate outline first, then fill in details
- Faster responses for long-form content

## 3.7 Prompt Evaluation & A/B Testing

- Evaluation metrics: Accuracy, relevance, format adherence
- Red-teaming: Testing for failure modes
- Prompt versioning: Track changes over time

### Evaluation Framework

# Simple prompt A/B test

    "v1": "Summarize this text in 3 bullet points.",

```

## 5.2 Plan Mode vs. Act Mode

### Toggling Between Modes

### Best Practice
> Always start complex work in Plan Mode. Review the plan carefully. Only switch to Act Mode after you understand and agree with the approach.

## 6.1 What is MCP?

**Key Concepts:**
- **Model Context Protocol (MCP)**: Open protocol for connecting AI assistants to external tools and data
- Created by Anthropic, now an open standard
- Three core primitives: **Tools**, **Resources**, and **Prompts**
- Client-server architecture
- Language-agnostic (works with TypeScript, Python, Go, Rust, etc.)
- The "USB-C for AI applications"

### Why MCP Matters

Before MCP, every AI tool integration was custom-built. MCP standardizes:
- How tools are exposed to AI models
- How resources (files, databases) are accessed
- How prompts are shared and reused
- Security and permission boundaries

📺 [MCP: Clearly explained — Anthropic/Community](https://www.youtube.com/watch?v=7pB7xPpP5pQ) (15:00) 🟡 📺
> **Why included:** Clear explanation of MCP architecture and why it matters for building extensible AI applications.

## 6.2 MCP Core Primitives

### Tools
- Functions that AI can call to perform actions
- Defined with JSON Schema (name, description, parameters)
- Examples: `read_file`, `search_web`, `send_email`, `query_database`

### Resources
- Read-only data sources that provide context
- Accessed via URI scheme
- Examples: `file://`, `db://`, `api://`, `git://`

### Prompts
- Reusable prompt templates
- Can be parameterized
- Shared between team members

### Server Architecture

```
┌─────────────────┐      ┌──────────────────┐
│   AI Assistant  │◄────►│   MCP Client     │
│  (Claude Code)  │      │  (Built into CC) │
└─────────────────┘      └────────┬─────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
               ┌────────┐  ┌──────────┐  ┌──────────┐
               │  Tool  │  │ Resource │  │  Prompt  │
               │ Server │  │  Server  │  │  Server  │
               └────────┘  └──────────┘  └──────────┘
```

## 6.3 Building MCP Servers (TypeScript & Python)

### TypeScript MCP Server

```typescript
// server.ts — A simple MCP server
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

const server = new Server({
  name: "my-tools",
  version: "1.0.0",
}, {
  capabilities: {
    tools: {},
  },
});

// Define available tools
server.setRequestHandler("tools/list", async () => {
  return {
    tools: [
      {
        name: "calculate",
        description: "Perform a mathematical calculation",
        inputSchema: {
          type: "object",
          properties: {
            expression: {
              type: "string",
              description: "Math expression to evaluate",
            },
          },
          required: ["expression"],
        },
      },
    ],
  };
});

// Handle tool execution
server.setRequestHandler("tools/call", async (request) => {
  if (request.params.name === "calculate") {
    const expr = request.params.arguments?.expression;
    const result = eval(expr); // In production, use a safe math parser!
    return {
      content: [{ type: "text", text: String(result) }],
    };
  }
  throw new Error("Unknown tool");
});

// Start the server
const transport = new StdioServerTransport();
await server.connect(transport);
```

### Python MCP Server

```python
# server.py — A simple MCP server in Python
from mcp.server import Server
from mcp.types import TextContent
import asyncio

app = Server("my-python-tools")

@app.list_tools()
async def list_tools():
    return [
        {
            "name": "greet",
            "description": "Generate a personalized greeting",
            "inputSchema": {
                "type": "object",
                "properties": {
                    "name": {"type": "string", "description": "Person's name"},
                    "language": {"type": "string", "description": "Language code"}
                },
                "required": ["name"]
            }
        }
    ]

@app.call_tool()
async def call_tool(name, arguments):
    if name == "greet":
        person_name = arguments.get("name")
        language = arguments.get("language", "en")
        greetings = {
            "en": f"Hello, {person_name}!",
            "es": f"Hola, {person_name}!",
            "fr": f"Bonjour, {person_name}!"
        }
        return [TextContent(type="text", text=greetings.get(language, greetings["en"]))]
    raise ValueError(f"Unknown tool: {name}")

if __name__ == "__main__":
    asyncio.run(app.run_stdio_async())
```

## 6.4 MCP Security & Sandboxing

**Key Concepts:**
- **Principle of least privilege**: Only expose necessary tools
- **Input validation**: Sanitize all arguments
- **No eval()**: Use safe parsers (e.g., `mathjs` instead of `eval`)
- **Authentication**: For sensitive tools, require auth tokens
- **Rate limiting**: Prevent abuse
- **Audit logging**: Track all tool calls

### Security Checklist

```markdown
- [ ] Input validation on all tool parameters
- [ ] No eval() or exec() — use safe parsers
- [ ] Authentication for sensitive operations
- [ ] Rate limiting implemented
- [ ] Audit logging enabled
- [ ] Error messages don't leak sensitive info
- [ ] Timeout on all operations
- [ ] Principle of least privilege for file access
```

**Key Concepts:**

- Bidirectional communication: Client → Server → Client

### Transport Options

- **WebSockets**: For real-time bidirectional communication

### Exercise 6.1: Weather MCP Server

Build an MCP server that provides weather information:

```typescript
// weather-server.ts
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

// Use a real weather API like OpenWeatherMap
const API_KEY = process.env.OPENWEATHER_API_KEY;

const server = new Server({
  name: "weather",
  version: "1.0.0",
}, { capabilities: { tools: {} } });

server.setRequestHandler("tools/list", async () => ({
  tools: [
    {
      name: "get_current_weather",
      description: "Get current weather for a city",
      inputSchema: {
        type: "object",
        properties: {
          city: { type: "string", description: "City name" },
          units: { type: "string", enum: ["metric", "imperial"], default: "metric" }
        },
        required: ["city"]
      }
    },
    {
      name: "get_forecast",
      description: "Get 5-day weather forecast",
      inputSchema: {
        type: "object",
        properties: {
          city: { type: "string", description: "City name" }
        },
        required: ["city"]
      }
    }
  ]
}));

server.setRequestHandler("tools/call", async (request) => {
  const { name, arguments: args } = request.params;
  
  if (name === "get_current_weather") {
    const response = await fetch(
      `https://api.openweathermap.org/data/2.5/weather?q=${args.city}&appid=${API_KEY}&units=${args.units}`
    );
    const data = await response.json();
    return {
      content: [{
        type: "text",
        text: `Weather in ${data.name}: ${data.weather[0].description}, ${data.main.temp} degrees`
      }]
    };
  }
  
  throw new Error("Unknown tool");
});

const transport = new StdioServerTransport();
await server.connect(transport);
```

## 12.1 Agent Fundamentals: Loops, Memory, Tools

**Key Concepts:**
- **Agent**: An AI system that perceives, reasons, and acts autonomously
- **Agent loop**: Observe → Think → Act → Repeat
- **Memory**: Short-term (session) and long-term (persistent)
- **Tools**: Functions the agent can call (APIs, databases, files)
- **Planning**: Breaking goals into sub-tasks
- **Reflection**: Self-evaluation and error correction

### The Agent Loop

```python
class Agent:
    def __init__(self, tools, memory):
        self.tools = tools
        self.memory = memory
    
    def run(self, goal, max_iterations=10):
        for i in range(max_iterations):
            # Observe: Gather context
            context = self.observe()
            
            # Think: Decide next action
            thought = self.think(goal, context)
            
            # Act: Execute the action
            action = self.decide_action(thought)
            result = self.execute(action)
            
            # Remember: Store in memory
            self.memory.store(action, result)
            
            # Check if goal achieved
            if self.goal_achieved(goal, result):
                return result
        
        return "Max iterations reached"
```

```python
# 1. Input sanitization
def sanitize_input(user_input):
    # Remove potential injection patterns
    blocked = ["ignore previous instructions", "system prompt"]
    for pattern in blocked:
        user_input = user_input.replace(pattern, "[REDACTED]")
    return user_input

# 2. Output filtering
class OutputFilter:
    def __init__(self):
        self.blocked_patterns = [
            r"\b\d{16}\b",  # Credit card numbers
            r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b",  # Emails
        ]
    
    def filter(self, output):
        for pattern in self.blocked_patterns:
            output = re.sub(pattern, "[REDACTED]", output)
        return output

# 3. Permission boundaries
class BoundedAgent:
    def __init__(self, allowed_tools):
        self.allowed_tools = allowed_tools
    
    def call_tool(self, tool_name, **kwargs):
        if tool_name not in self.allowed_tools:
            raise PermissionError(f"Tool {tool_name} not allowed")
        return self.allowed_tools[tool_name](**kwargs)
```

## Appendix B: API Key Setup

### B.1 Anthropic API Key

1. Go to https://console.anthropic.com
2. Sign up / Sign in
3. Navigate to "API Keys"
4. Click "Create Key"
5. Copy and store securely
6. Set as environment variable:
```bash
export ANTHROPIC_API_KEY="sk-ant-your-key-here"
# Add to ~/.bashrc or ~/.zshrc for persistence
```

### B.2 OpenAI API Key

1. Go to https://platform.openai.com
2. Sign up / Sign in
3. Navigate to "API Keys" in settings
4. Click "Create new secret key"
5. Copy and store securely (shown only once!)
6. Set as environment variable:
```bash
export OPENAI_API_KEY="sk-your-key-here"
# Add to ~/.bashrc or ~/.zshrc
```

### B.3 Google AI (Gemini) API Key

1. Go to https://aistudio.google.com
2. Sign in with Google account
3. Click "Get API Key"
4. Create new key or use existing
5. Set as environment variable:
```bash
export GEMINI_API_KEY="your-key-here"
```

### B.4 Kimi API Key

1. Go to https://platform.moonshot.cn
2. Sign up / Sign in
3. Navigate to API Keys section
4. Create new key
5. Set as environment variable:
```bash
export KIMI_API_KEY="your-key-here"
```

### B.5 Security Best Practices

- Never commit API keys to version control
- Use environment variables or secret managers
- Rotate keys regularly (every 90 days)
- Use different keys for development and production
- Monitor usage for unexpected spikes
- Set spending limits where available
- Use `.env` files with `.env` in `.gitignore`

```bash
# .env file (NEVER commit this!)
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
GEMINI_API_KEY=...
KIMI_API_KEY=...

# .gitignore
.env
.env.local
*.key
```

## Appendix D: Glossary

| Term | Definition |
|------|------------|
| **AI (Artificial Intelligence)** | Computer systems that can perform tasks typically requiring human intelligence |
| **Agent** | An AI system that perceives its environment and takes autonomous actions |
| **AGI (Artificial General Intelligence)** | Hypothetical AI with human-like general intelligence |
| **API (Application Programming Interface)** | A way for software applications to communicate |
| **Artifact** | Interactive output window in Claude for code, documents, visualizations |
| **Attention Mechanism** | Neural network component that allows models to focus on relevant parts of input |
| **Backpropagation** | Algorithm for training neural networks by adjusting weights |
| **Chain-of-Thought (CoT)** | Prompting technique where the model shows its reasoning step-by-step |
| **CLAUDE.md** | Project configuration file for Claude Code |
| **Context Window** | The maximum amount of text an LLM can process at once |
| **Deep Learning** | Machine learning using neural networks with many layers |
| **Embedding** | Numerical representation of text in a high-dimensional vector space |
| **Epoch** | One complete pass through the training dataset |
| **Few-Shot Prompting** | Providing examples in the prompt to guide model behavior |
| **Fine-tuning** | Adapting a pre-trained model to a specific task with additional training |
| **Hallucination** | When an AI generates plausible but incorrect or fabricated information |
| **HITL (Human-in-the-Loop)** | Systems where humans review or approve AI decisions |
| **Inference** | The process of generating outputs from a trained model |
| **LLM (Large Language Model)** | AI model trained on vast text data to understand and generate language |
| **MCP (Model Context Protocol)** | Open protocol for connecting AI assistants to external tools |
| **Neural Network** | Computing system inspired by biological neural networks |
| **Prompt** | Input text that instructs an AI model what to do |
| **Prompt Engineering** | The practice of designing effective prompts for AI models |
| **RAG (Retrieval Augmented Generation)** | Technique combining information retrieval with text generation |
| **ReAct (Reasoning + Acting)** | Pattern where LLMs alternate between reasoning and taking actions |
| **Reinforcement Learning** | Training method where agents learn through trial and error with rewards |
| **RLHF (Reinforcement Learning from Human Feedback)** | Fine-tuning models using human preference data |
| **Sandbox** | Isolated environment for safely executing code |
| **SKILL.md** | Claude Code skill definition file for reusable workflows |
| **Subagent** | Separate Claude Code instance for isolated tasks |
| **Token** | Unit of text processing in LLMs (roughly 0.75 words) |
| **Transformer** | Neural network architecture that powers modern LLMs |
| **Zero-Shot** | Tasking an AI without providing examples |
