## 7.8 Security & Sandboxing

**Key Concepts:**

- **Kernel-level sandboxing**: Code runs in isolated environments

- Automatic dependency detection and installation

- Network access controls

- File system restrictions

- Sandboxing prevents malicious code execution

- Critical for running untrusted code

### Security Features

| Feature | Description |

|---------|-------------|

| Isolated Environment | Each session runs in a fresh container |

| Network Controls | Configurable internet access |

| File Restrictions | Limited to project directory |

| Dependency Scanning | Automatic vulnerability checks |

| Execution Timeout | Prevents runaway processes |

### GitHub Actions Example

```yaml
# .github/workflows/codex-ci.yml
name: Codex CI
on: [push, pull_request]
jobs:
  codex-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Codex CLI
        run: npm install -g @openai/codex
      - name: Run Codex Review
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: codex review --fail-on-issues
```
