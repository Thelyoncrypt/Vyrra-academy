## Lesson

The transformer architecture consists of:

Tokenization: Text split into subword tokens (BPE or SentencePiece)

Embedding Layer: Each token ID mapped to a vector (vocab_size x d_model)

Positional Encoding: Adds position information (RoPE in modern models)

Transformer Blocks: Each contains RMSNorm, Grouped Query Attention with causal mask and KV cache, and SwiGLU Feed-Forward Network

Output Head: Projects to vocabulary size; softmax over possible next tokens

The Attention Mechanism: Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V. Query represents "what am I looking for?", Key represents "what do I contain?", and Value represents "what information do I have?". Multi-Head Attention computes multiple attention maps in parallel, each specializing in different relationship types (adjective-noun pairs, subject-verb, long-distance dependencies).

Inference Phases: 1. Prefill Phase: Process entire prompt (compute-bound). Decode Phase: Generate one token at a time using KV cache (memory-bandwidth-bound)
