## 10.3 Cost Analysis & Optimization

### Pricing Comparison (May 2026)

| Tool | Free Tier | Paid Tier | Enterprise |

|------|-----------|-----------|------------|

| Claude Code | 5x queries/day | $20/mo Pro | $30/mo Team |

| Codex | Limited usage | $20/mo ChatGPT Plus | Custom |

| Copilot | 2K completions/mo | $10/mo Individual | $19/mo Business |

| Cursor | 500 fast requests | $20/mo Pro | Custom |

### Cost Optimization Strategies

1. **Use the right model for the task**: Don't use Opus for simple tasks

2. **Clear context regularly**: `/clear` when switching tasks

3. **Use CLAUDE.md/AGENTS.md**: Better context = fewer iterations

4. **Batch similar tasks**: Group related changes

5. **Plan before acting**: Reduces rework

6. **Use jCodeMunch MCP**: Indexes codebases, reduces token burn

## 10.4 When to Use Which Tool

### Decision Tree

```text
Are you working in an IDE primarily?
├── Yes → Cursor or Copilot
│   ├── Need agentic capabilities? → Cursor
│   └── Want seamless integration? → Copilot
└── No → Terminal-based
    ├── Need secure code execution? → Codex
    ├── Building reusable workflows? → Claude Code
    └── General purpose with best reasoning? → Claude Code
```

### By Task Type

| Task | Best Tool | Why |

|------|-----------|-----|

| Quick bug fix | Copilot or Cursor | Fastest |

| Complex refactor | Claude Code | Plan mode, best reasoning |

| Untrusted code | Codex | Sandboxing |

| Team standardization | Claude Code | Skills, CLAUDE.md |

| Vibe coding | Cursor | Fastest iteration |

| Enterprise deployment | Claude Code or Codex | Security, governance |

| Learning/exploring | Claude Code | Best explanations |

## 10.5 Token Management & Cost Control

### Understanding Token Economics

- **Input tokens**: Everything you send (files, messages, context)

- **Output tokens**: Everything the model generates

- **Cost varies by model**: Opus > Sonnet > Haiku (for Claude)

- **Context accumulation**: Every turn adds to the conversation

### Token Management Strategies

```bash
# Monitor current usage
/context          # Claude Code

# Clear context when switching tasks
/clear

# Use smaller models for simple tasks
/model sonnet     # Instead of opus for simple tasks

# Limit file reads — only include what's needed
# Use .claudeignore to exclude large/generated files

# Compress CLAUDE.md — concise instructions cost less
```
