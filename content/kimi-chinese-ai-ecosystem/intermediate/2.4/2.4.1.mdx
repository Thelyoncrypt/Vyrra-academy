## 2.2 Claude Ecosystem Overview

**Key Concepts:**

- **Claude.ai**: Web interface for general use (Free, Pro, Team plans)

- **Claude Code**: Terminal-based AI coding agent (developer tool)

- **Claude Cowork**: Non-technical automation platform

- **Claude API**: Programmatic access for building applications

- **Model tiers**: Haiku (fast), Sonnet (balanced), Opus (powerful), Extended Thinking

- **Key strengths**: Reasoning, coding, safety, long context (1M+ tokens)

- **Unique features**: Artifacts, Projects, Memory, Skills, MCP integration

## 2.3 OpenAI/Codex Ecosystem Overview

**Key Concepts:**

- **ChatGPT**: Consumer-facing chat interface

- **Codex**: AI coding agent (CLI, web, IDE, GitHub integration)

- **GPT-5.x series**: Latest frontier models

- **OpenAI API**: Comprehensive API platform

- **Codex features**: Sandboxed execution, multi-file projects, plan mode, AGENTS.md

- **Key strengths**: Ecosystem breadth, enterprise adoption, GitHub integration

- **Security**: Kernel-level sandboxing for code execution

## 2.4 Google AI (Gemini) Ecosystem Overview

**Key Concepts:**

- **Gemini models**: 2.5 Pro, 2.5 Flash, Flash-Lite tiering

- **Google AI Studio**: Free playground for experimentation

- **Gemini in Workspace**: Docs, Sheets, Gmail, Slides integration

- **NotebookLM**: Source-grounded research tool

- **Imagen 4 & Veo 3.1**: Image and video generation

- **Vertex AI**: Enterprise AI platform

- **AntiGravity**: Google's AI web design tool

- **Key strengths**: Integration with Google services, multimodal capabilities, free tier generosity

## 2.5 Kimi Ecosystem Overview

**Key Concepts:**

- **Kimi K2.5/K2.6**: Moonshot AI's frontier models

- **Kimi Code**: Coding-focused CLI tool

- **Agent Swarm capabilities**: PARL training framework

- **Open-weights availability**: Self-hosting options

- **Key strengths**: Long context (2M+ tokens), agentic capabilities, open-weights

- **Positioning**: The open alternative for agent builders

## 2.6 Open-Source & Hermes Agent

**Key Concepts:**

- **Hermes Agent**: Open-source autonomous agent framework

- **Open-source models**: Qwen, GLM, DeepSeek, Llama

- **Self-hosting**: Running models locally with Ollama, LM Studio, vLLM

- **MCP ecosystem**: 10,000+ community-built servers

- **When to choose open-source**: Cost control, privacy, customization

## 2.7 Choosing the Right Tool for the Job

**Decision Framework:**

| Need | Best Tool | Why |

|------|-----------|-----|

| General chat/research | Claude.ai or ChatGPT | Best reasoning, most helpful |

| Coding (serious projects) | Claude Code or Codex | Agentic workflows, MCP support |

| Google Workspace user | Gemini | Native integration |

| Long documents (2M+ tokens) | Kimi | Largest context window |

| Cost-sensitive | Open-source + self-host | No API fees |

| Image generation | GPT Image 2, Midjourney v7, Imagen 4 | Best quality |

| Video generation | Seedance 2.0, Veo 3.1, Kling 3 | Leading models |

| Automation/workflows | Claude Cowork, n8n, Make | Ecosystem depth |
