# Semantic Caching and Prompt Caching Strategies

## Lesson

Semantic caching uses vector embeddings to match queries with similar meaning despite different wording, delivering cache hit rates of 60-85% in FAQ and support workloads with 96.9% latency reduction for cached queries (1.67s to 0.052s) and up to 73% cost reduction in conversational workloads.

# Semantic caching patternfrom sentence_transformers import SentenceTransformerimport chromadbmodel = SentenceTransformer('all-MiniLM-L6-v2')client = chromadb.Client()cache = client.create_collection("semantic_cache")def cached_completion(query: str, threshold: float = 0.92) -> str: query_embedding = model.encode(query) results = cache.query(query_embeddings=[query_embedding], n_results=1) if results['distances'][0][0] > threshold: return results['documents'][0][0] # Cache hit # Cache miss: call LLM, store result response = llm.complete(query) cache.add(embeddings=[query_embedding], documents=[response], ids=[uuid()]) return response

Prompt Caching (Token-Level): Place static content (system prompts, context documents, few-shot examples) early in the prompt sequence. Cached input tokens cost approximately 10% of normal input tokens. The critical technique: static content first, dynamic content last.
