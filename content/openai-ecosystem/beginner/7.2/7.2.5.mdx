## Lesson

Three Fine-Tuning Methods (2026): SFT (Supervised Fine-Tuning) for demonstrable behaviors with 50-500 examples on GPT-4.1/GPT-4.1-mini; DPO (Direct Preference Optimization) for ranking pairs; and RFT (Reinforcement Fine-Tuning) for verifiable-correct tasks requiring a programmable grader on o4-mini only. GPT-5.4 family is NOT available for fine-tuning.

Critical caveat: Fine-tuning does NOT improve safety. SFT on bad data makes models less safe. Add evals for jailbreaks and harmful output to post-tune validation. GPT-4.1-mini training runs ~$5-$50 for typical SFT jobs; below ~1M calls/month, a stronger prompt is usually cheaper.

Evals API provides automated evaluation with custom graders. Define test datasets, create graders (string_check, text_similarity, multi_select, or custom LLM-based), and run evals programmatically. The Prompt Optimizer dashboard optimizes prompts based on production failures.

Production Deployment (Azure OpenAI): Implement rate limiting (per-user and per-application quotas), anomaly alerts (usage spikes, elevated error rates, latency increases), horizontal scaling with load balancing, and caching for frequently accessed data.

Open-Weight Deployment (gpt-oss): Apache 2.0 licensed; two sizes: gpt-oss-120b (80GB H100 memory) and gpt-oss-20b (16GB, local/edge inference). Deployment frameworks: vLLM (production), Ollama (consumer), llama.cpp, LM Studio, Transformers, Together AI, Fireworks, Databricks, Cloudflare.

Critical Deadlines: Assistants API shutdown: August 26, 2026; Sora API shutdown: September 24, 2026; Fine-tuning platform winding down for new users.
