# App Design Fundamentals

> This module introduces core principles for designing mobile applications: touch-friendly interfaces, navigation patterns, onboarding flows, and platform-specific considerations for iOS and Android.

## Key Concepts

- Touch Targets
- Navigation Patterns
- Onboarding Best Practices
- Typography Baseline
- Cross-Platform Considerations

## Lesson

This module introduces core principles for designing mobile applications: touch-friendly interfaces, navigation patterns, onboarding flows, and platform-specific considerations for iOS and Android. Research shows that Jetpack Compose is now the default for ~70% of new Android apps, while SwiftUI accounts for ~65% of new iOS apps.

Native vs. Cross-Platform Decision Framework | Factor | Native | Cross-Platform | |---|---|-----| | Performance | Maximum | Near-native for 90% of business apps | | Initial cost | Higher | 30-40% lower for MVP | | Time to market | Slower | 30-50% faster | | Maintenance | Complex (two codebases) | Simpler (one fix, both platforms) | | Best for | Hardware-intensive apps | MVPs, content apps, business tools |

## Practice

### Navigation Pattern Analysis

Navigation Pattern Analysis (45 min): Download 5 popular apps (2 native iOS, 2 native Android, 1 cross-platform). Document navigation model, touch target sizes, and onboarding flow for each.

### Wireframe an Onboarding Flow

Wireframe an Onboarding Flow (60 min): Design a 3-screen onboarding flow for a habit-tracking app in Figma. Include value proposition, permission request timing, and account creation.

### Platform Comparison

Platform Comparison (30 min): Compare the same screen (a settings page) between iOS and Android native apps. Document differences in typography, spacing, navigation, and interaction patterns.
